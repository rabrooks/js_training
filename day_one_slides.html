var statment

  • Declares and initializes variables within a function
  • A variable declared anywhere within a function is visible everywhere within the function
  • No block scope in javascript

The way javascript handles variables is unique, the var statment is treated as two parts

First the declaration of the var gets hoisted to the top of the function and set to undefined

Second the initiliztion becomes an assignment

Rules To Live By

  • Declare all variables at the top of the function.
  • Declare all functions before you call them.


The language provides mechanisms that allow you to ignore this advice, but they are problematic.

Unlike other C syntax languages that have block scope, you should declare all variables at the top of the function

Arguments


  • When a function is invoked, in addition to its parameters, it also gets a special parameter called arguments.
  • It contains all of the arguments from the invocation.
  • It is an array-like object.
  • Arguments.length is the number of arguments passed.
  • Rule of thumb treat this as a read only structure

Arguments Example



function sum() {
    var i,
        n = arguments.length,
        total = 0;
    for (i = 0; i < n; i += 1) {
        total += arguments[i];
    }
    return total;
}

var ten = sum(1, 2, 3, 4);

this

  • The this parameter contains a reference to the object of invocation.
  • this allows a method to know what object it is concerned with.
  • this allows a single function object to service many objects.
  • this is key to prototypal inheritance.

If the this parameter does not make sense right now, it will once we start working with methods and inheritance

Invoking Functions

Javascript functions can be invoked in four different ways

  • functions
  • methods
  • constructors
  • indirectly using call() or apply() methods on the function

The way we call functions, will determine what happens to the this parameter

Function Invocation

Functions are invoked with an invocation expression, that evaluates to a function object


function logProps(prop) {
    console.log(prop)
}

var foo = "bar"

logProps(foo);
//Function Invocation with and object
logProps({x:1});

When a function is called in the function form, this is set to the global object. That is not very useful.

Function Invocation


function foo() {
    var that = this;

    function() {
        console.log(this);
        console.log(that);
    }
}

An inner function does not get access to the outer this. var that = this;

Constructor Invocation

A constructor invocation creates a new, empty object that inherits from the prototype property of the constructor.


var bar = new Foo();
  // or
var bar = new Foo;

The new object that is created gets assigned to this.

Notice, when using the Constructor Invocation we can ignore the () suffix, if we are not passing any parameters

Indirect Invocation (Apply Form)

Functions are objects and objects have methods. Two of these methods, call() and apply(), invoke the function indirectly.




bar.call(foo, 1, 2);

bar.apply(foo, [1,2]); 
// apply expects an array for the argument list

Both methods allow you to specify the this value

Here we are calling the function object bar and setting the this value to be the foo object
